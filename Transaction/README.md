# Проект з Блокчейн Технологій

## Опис проекту

Це навчальний проект, який демонструє основні концепції блокчейн-технологій, включаючи:
- Створення криптографічних гаманців
- Цифрові підписи транзакцій
- Mempool (пул очікуючих транзакцій)
- Майнінг блоків (Proof of Work)
- Валідація транзакцій та блоків

## Структура проекту

```
Transaction/
├── Models/
│   ├── Wallet.cs                  # Гаманець з криптографічними ключами
│   ├── BlockchainTransaction.cs   # Транзакція в блокчейні
│   ├── Mempool.cs                 # Пул неопрацьованих транзакцій
│   └── Block.cs                   # Блок блокчейну
├── Program.cs                     # Головна програма-демонстрація
└── Transaction.csproj             # Файл проекту
Theory.md                          # Детальна теорія роботи блокчейну
README.md                          # Цей файл
```

## Що демонструє програма

Програма виконує наступні кроки:

### 1. Створення гаманців
- Генерація приватних та публічних ключів за допомогою RSA
- Створення адрес гаманців на основі хешування публічних ключів

### 2. Створення та підпис транзакцій
- Формування транзакцій між користувачами
- Цифровий підпис транзакцій приватними ключами
- Демонстрація даних до і після підпису

### 3. Робота з Mempool
- Додавання транзакцій в пул очікування
- Валідація транзакцій перед додаванням
- Показ статистики mempool

### 4. Перевірка валідності
- Перевірка цифрових підписів
- Валідація даних транзакцій
- Захист від невалідних операцій

### 5. Майнінг блоків
- Вибір транзакцій з найвищими комісіями
- Створення винагороди майнеру (coinbase transaction)
- Процес майнінгу (підбір Nonce для знаходження правильного хешу)

### 6. Валідація блоків
- Перевірка всіх транзакцій в блоці
- Перевірка правильності хешу
- Перевірка складності майнінгу

### 7. Захист від шахрайства
- Демонстрація відхилення невалідних транзакцій
- Захист від підпису чужих транзакцій

## Основні класи

### Wallet (Гаманець)
Клас для роботи з криптографічними ключами:
- Генерація пари ключів (приватний/публічний)
- Підпис даних приватним ключем
- Перевірка підписів публічним ключем

### BlockchainTransaction (Транзакція)
Представляє переказ коштів:
- Адреси відправника та отримувача
- Сума та комісія
- Цифровий підпис
- Методи валідації

### Mempool (Пул транзакцій)
Зберігає неопрацьовані транзакції:
- Додавання нових транзакцій
- Вибір транзакцій за комісією
- Статистика та управління

### Block (Блок)
Контейнер для транзакцій:
- Список транзакцій
- Зв'язок з попереднім блоком
- Процес майнінгу (Proof of Work)
- Валідація блоку

## Додаткова інформація

Цей проект створено виключно в навчальних цілях для демонстрації основних принципів роботи блокчейну. Він є спрощеною версією справжніх блокчейн-систем та не призначений для використання в продакшені.

### Відмінності від реальних блокчейнів:

- Відсутня P2P мережа (все працює локально)
- Немає персистентного зберігання блоків
- Спрощена система валідації
- Низька складність майнінгу (для швидкої демонстрації)
- Відсутня система консенсусу між вузлами

### Подальші покращення:

Для більш реалістичної системи можна додати:
- Збереження блокчейну в базу даних
- P2P мережу для розповсюдження транзакцій
- Механізм консенсусу (наприклад, longest chain rule)
- Підтримку UTXO моделі (Unspent Transaction Output)
- Більш складні правила валідації
- Wallet файли з шифруванням
